Abstract

The ontology for the Annotation of Push-Notifications (APN) aims to provide a tool that can be used to model different aspects of push-notifications, i.e., communication tools used by apps and services to disseminate information, engage with their users and encourage users to take particular actions. The scope of this ontology is limited to the definition of a set of parameters that can be used to classify push-notification data, in particular its text content, with the end goal of creating a service which pushes more empathetic content to push-notification subscribers.

2. Introduction

2.1 Ontology Requirement Specification Document

Ontology for the Annotation of Push-Notifications
1. Purpose
The purpose of this ontology is to support the annotation and extraction of insights from push-notification data.
2. Scope
The scope of this ontology is limited to the definition of a set of parameters that can be used to classify push-notification data, in particular its text content, with the end goal of creating a service which pushes more empathethic content to push-notification subscribers. Specifically, the notification content will be used to categorize the notification in terms of: (i) its spec'ific call-to-action, (ii) and respective marketing campaign type, (iii) its target audience, (iv) its time sensitiveness and (v) the presence of information that can be used to infer personal data from the subscribers.
3. Implementation Language
RDF
4. Intended End-Users
Organizations with Web or application services which include push-notifications in their communication flow with customers.
5. Intended Uses
Use 1. Creation of multi-label classification models to automatically annotate push-notifications.
Use 2. Generation of push-notification content based on annotations' model.
6. Ontology Requirements
a. Non-Functional Requirements
NFR 1. The ontology shall be published online with standard documentation.
b. Functional Requirements: Groups of Competency Questions
CQG1. Related to push-notification services CQG2. Related to push-notification categorization
CQ1. What is the content of the push-notification?
CQ2. Which service / application pushed the notification?
CQ3. What is the application type in relation to its content? 		CQ4. What is the call-to-action (CTA) of the notification?
CQ5. Does the notification belong to a specific campaign type?
CQ6. How urgent is the notification?
CQ7. Does the notification have a clear target audience?
CQ8. Does the notification contain features which indicate personalization?

3. Base ontology

3.1 Push Notification

Definition: Push Notification
IRI: https://w3id.org/apn#PushNotification
Description: Push notifications are pop-up messages, sent by mobile or browser applications, that appear on the users‚Äô devices. Subscribers can be anywhere and still receive these messages as long as they‚Äôre online or have their browsers running on their devices.
Definition: notification ID
IRI: https://w3id.org/apn#notificationID
Description: Unique identifier of the notification
Domain: PushNotification
Range: xsd:integer
Definition: notification text
IRI: https://w3id.org/apn#notificationText
Description: Text content of the notification, including emojis
Domain: PushNotification
Range: xsd:string
Definition: pushed on
IRI: https://w3id.org/apn#pushedOn
Description: Date and time when the notification is pushed by the application
Domain: PushNotification
Range: xsd:dateTime
Definition: pushed by
IRI: https://w3id.org/apn#pushedBy
Description: Indicates the app that pushed the notification
Domain: PushNotification
Range: Application

3.2 Application

Definition: Application
IRI: https://w3id.org/apn#Application
Description: Computer program or software application designed to run on a mobile device such as a phone, tablet, or watch. Mobile applications often stand in contrast to desktop applications which are designed to run on desktop computers, and web applications which run in mobile web browsers rather than directly on the mobile device.
Definition: app name
IRI: https://w3id.org/apn#appName
Description: The name of the application
Domain: Application
Range: xsd:string
Definition: has app type
IRI: https://w3id.org/apn#hasAppType
Description: Indicates the application type
Domain: Application
Range: ApplicationType

3.3 Urgency

Definition: Urgency
IRI: https://w3id.org/apn#Urgency
Description: The urgency of the notification - if it contains information that is only relevant for a certain period of time and becomes redundant if that period of time passes (for example, a notification about Friday night is irrelevant if delivered on Saturday morning).
Superclass Of: NextMinutes, NextHours, NextDays, NextWeeks, NextMonths, NextYears, Holidays, Seasons, Unclear
Definition: is time sensitive
IRI: https://w3id.org/apn#isTimeSensitive
Description: Indicates weather the notification is urgent, i.e. time-sensitive, or not.
Domain: PushNotification
Range: xsd:boolean
Definition: remains relevant
IRI: https://w3id.org/apn#remainsRelevant
Description: Indicates the urgency of the notification - the period of time when the notification remains relevant for the user
Domain: PushNotification
Range: Urgency

3.4 Call-To-Action

Definition: Call-To-Action
IRI: https://w3id.org/apn#CallToAction
Description: The call-to-action of the notification - for example, "Anne sent you a message on App X" is encouraging the user to interact with the app to communicate with other people.
Superclass Of: AccountManagement, Awareness, Claim, Discover, Download, Feedback, Play, Purchase, Reminder, Social, Watch

3.5 Personalisation

Definition: Personalisation
IRI: https://w3id.org/apn#Personalisation
Description: Indicates personal information that can be derived or inferred from the notification.
Superclass Of: Age, Authenticating, BrowsingBehavior, DeviceBased, EthnicOrigin, FamilyStructure, Location, MedicalHealth, PoliticalAffiliation, Interest, PersonalMotivations, Professional, PurchasesAndSpendingHabit, SocialNetwork

3.6 Target Audience

Definition: Target Audience
IRI: https://w3id.org/apn#TargetAudience
Description: Indicates the audience being targeted by the notification, based on their behavior or other demographic, geographic or psychographic characteristics.
Superclass Of: Behavior, Demographic, Geographic, Psychographic

3.7 Campaign Type

Definition: Campaign Type
IRI: https://w3id.org/apn#CampaignType
Description: Indicates the campaign type of the notification, based on the marketing strategy followed by a company to keeps its brand's visibility, i.e., to mantain and increase the number of notification subscribers.
Superclass Of: Awareness, Breaking Content, Help & Feedback & Tips, Loyalty, Newsletter, Onboarding, Product, Promotional, Re-engagement, Social

4. Urgency categories

Notifications can be classified in terms of its urgency, i.e., if they contain information that is only relevant for a certain period of time and becomes redundant after it (for example, a notification about Friday night is irrelevant if delivered on Saturday morning).

Examples:

Hungryüë∂? You know what to doüôãÔ∏è üôã Ô∏èGrab the best deals on babyüçº food processors, breast pumps, pacifiers, other must-haves Hurry! Up to 70% off feeding essentials

In the notification above, the content includes words like "hurry" and "70% off" suggesting an offer may expire soon, however the urgency of the notification is unclear.

Flurv üí¨ Henrybrett just joined our local community - Be the first to welcome him!

In the notification above, "Henrybrett" is a new member of a social community and needs to feel welcome - while not a very urgent notification, this guy will not be a new member in a few weeks, therefore this notification is relevant for just a few days before it is redundant.

5. Call-To-Action categories

Notifications can be classified in terms of its call-to-action, i.e., if they encourage subscribers to perform a certain type of action.

Examples:

LAST CHANCE -50% üî• 600+ styles üõí Shop before midnight >>

In the notification above, the call-to-action encourages the subscribers make a purchase.

What the Delta variant so much more contagious, and what does it mean for your life, whether you're vaccinated or unvaccinated? Listen now

In the notification above, the call-to-action teases the subscribers to interact with the app to discover more about the topic.

The defined call-to-action subclasses are shown in the figure below.

5.1 Account Management

Definition: Account Management
IRI: https://w3id.org/apn#AccountManagement
Description: Notifications that encourage subscribers to create and/or log into an account to take advantage of a certain feature.

5.2 Awareness

Definition: Awareness
IRI: https://w3id.org/apn#Awareness
Description: The notification makes subscribers aware of information that does not need further investigation by the subscriber, e.g., a new feature, upcoming holiday, weather, stock price, score in a match, etc.

Definition: Claim

Description: Notifications that encourage the subscribers to click and claim some prize or reward from the application.

5.4 Discover

Definition: Discover

5.5 Download

5.6 Feedback

Definition: Feedback
IRI: https://w3id.org/apn#Feedback

5.7 Play

Definition: Play
IRI: https://w3id.org/apn#Play
Description: Notifications related with gaming apps, where the subscriber is encouraged to re-engage with the game.

5.8 Purchase

Definition: Purchase
IRI: https://w3id.org/apn#Purchase
Description: Notifications to get the subscriber to purchase a product or service.

5.9 Reminder

Definition: Reminder
IRI: https://w3id.org/apn#Reminder
Description: Notifications that serve as a reminder or alert to the users or provide advice / tips.

5.10 Social

Definition: Social
IRI: https://w3id.org/apn#Social
Description: Notifications that encourage the subscriber to engage with social tools e.g. share, like, comment, reply.

5.11 Watch

Definition: Watch
IRI: https://w3id.org/apn#Watch
Description: Notifications that encourage the user to watch some content on the application.
